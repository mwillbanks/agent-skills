# Service Boundary Definition

Use this to define clean service boundaries and ownership.

## Prompt

Define service boundaries that optimize maintainability and clarity.

Requirements:
1. Identify bounded contexts and ownership lines.
2. Define API contracts between services.
3. Define data ownership and consistency strategy.
4. Minimize tight coupling and hidden cross-service dependencies.
5. Describe migration path from current state to target boundaries.

Output:
- Proposed boundaries
- Contract summary
- Coupling risks
- Migration plan

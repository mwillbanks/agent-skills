# Data Pipeline Design

Use this for designing ingestion, transformation, storage, and serving pipelines.

## Prompt

Design a robust data pipeline with production constraints.

Requirements:
1. Define input sources, schemas, and validation strategy.
2. Define transformation stages and idempotency guarantees.
3. Define storage model and retention lifecycle.
4. Define backfill, replay, and failure handling behavior.
5. Define monitoring, SLOs, and alerting points.

Output:
- End-to-end pipeline architecture
- Schema and contract strategy
- Reliability/failure model
- Operational model

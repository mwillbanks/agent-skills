# Documentation Update

Use this when implementation changed and docs must be synchronized immediately.

## Prompt

Update documentation to reflect actual implementation behavior.

Requirements:
1. Identify docs affected by code changes.
2. Update behavior, contracts, configuration, and usage examples.
3. Remove stale or contradictory statements.
4. Keep docs concise and developer-usable.
5. Confirm docs now match the code paths changed.

Output:
- Docs updated
- Why each update was necessary
- Remaining doc gaps (if any)

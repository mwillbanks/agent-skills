# Documentation Audit

Use this for a strict audit of documentation accuracy and coverage.

## Prompt

Audit docs against current code and report mismatches.

Requirements:
1. Compare docs claims to actual implementation.
2. Flag stale behavior, missing sections, and misleading examples.
3. Prioritize mismatches by impact.
4. Propose concrete fixes for each issue.
5. Apply safe doc fixes directly when obvious.

Output:
- High/Medium/Low doc issues
- Fix recommendations
- Safe updates applied

# Root Cause Analysis

Use this when a bug must be explained and fixed at its real cause.

## Prompt

Perform root-cause analysis and implement a verified fix.

Requirements:
1. Reproduce the issue with a clear trigger path.
2. Separate symptoms from root cause.
3. Prove the root cause with concrete evidence.
4. Implement fix at the proper layer.
5. Add regression protection (tests or checks).
6. Validate the original failure no longer reproduces.

Output:
- Reproduction steps
- Root cause evidence
- Fix details
- Regression protection
- Validation results

# Dependency Upgrade

Use this for controlled upgrades with low regression risk.

## Prompt

Perform a dependency upgrade with safety and validation discipline.

Requirements:
1. Scope upgrades explicitly (target packages and versions).
2. Analyze release notes and breaking-change risk.
3. Apply upgrades incrementally when risk is high.
4. Run full validation after upgrades.
5. Report compatibility changes and required follow-ups.

Output:
- Upgraded dependencies
- Breaking changes evaluated
- Validation results
- Required code/docs follow-ups

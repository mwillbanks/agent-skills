# Code Review Checklist

- [ ] Active runtime path traced in code and config.
- [ ] Public contract changes are intentional and tested.
- [ ] Security and tenancy boundaries are preserved.
- [ ] Failure modes have bounded retries/timeouts.
- [ ] Async work has idempotency and visibility.
- [ ] Observability covers new failure modes.
- [ ] Infrastructure and app code agree.
- [ ] Tests validate behavior without over-mocking.

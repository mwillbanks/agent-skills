# Repository Constitution

## Purpose

This repository exists to publish reusable agent skills that improve execution quality, specification discipline, validation honesty, and maintainability across agent-driven work.

## Standing Rules

1. Skill changes must remain generic unless the repository intentionally publishes a repo-specific skill.
2. Implementation-oriented skills must define enforceable done criteria, validation posture, and artifact expectations.
3. Spec-driven work must preserve history. Requirement amendments update existing specification artifacts instead of replacing them and erasing prior completed work.
4. Independent review gates must remain independent. Manager or worker narratives must not replace governing intent, specification artifacts, or repo state in review packets.
5. Durable execution artifacts must stay truthful. A reported status, plan, task list, or review state must match the actual repository state before completion is claimed.
6. When frontend guidance is published, it must prefer design-system, token, and composition patterns over hardcoded one-off implementations.

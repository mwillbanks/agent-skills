# Specification Workflow

This repository uses `.specify/` for feature-governed work that should keep specification, planning, validation, review, traceability, and post-mortem artifacts together.

## Rules

1. Update existing feature artifacts in place when requirements change.
2. Do not replace spec, plan, or tasks files in a way that erases prior completed context.
3. Keep review, validation, findings closure, traceability, and post-mortem artifacts in the active feature folder.
4. Treat `.specify/` as the primary truth surface for governed feature delivery.
